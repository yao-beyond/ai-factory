# 使用指南（給操作者）

這份文件寫給**不需要懂程式**的人。你只要會「提需求」和「看結果」，AI Factory 會幫你把程式寫好、檢查好，最後交給工程師按一個鍵合併。

---

## 它在幫你做什麼？

你提出一個需求，AI Factory 會自動：

1. 🧠 想好怎麼做（產生實作計畫）
2. 📝 **先讓你確認方向**（用白話說明它打算怎麼做，你按「確認開工」或「取消」）
3. 👩‍💻 派幾個 AI 工程師同時嘗試不同寫法
4. 🧪 挑出最好的一版
5. 🔍 自動做程式碼品質與安全檢查
6. 🛠️ 依檢查結果修正
7. ✅ 交付成果

整個過程你都能在一個網頁上看到進度，不用看任何程式碼。

**兩種使用方式（首頁可選）**

- ✨ **做一個全新的東西**：你只有一個點子、還沒有任何程式碼。**不需要 git 帳號或金鑰**，完成後直接給你一份可下載的成果；若做的是網頁，還能直接「線上預覽」。
- 🔧 **改現有的專案**：針對既有程式庫，完成後 AI 會開一個 **Pull Request（草稿）** 給工程師審查合併。

---

## 第一次使用：用網頁提需求（最簡單）

請你的管理員先完成 [SETUP.md](SETUP.md) 的安裝，然後：

1. 打開首頁（管理員會給你網址，通常是 `http://localhost:8080`）。
2. 選一種方式：**✨ 做一個全新的東西** 或 **🔧 改現有的專案**。
3. 填寫表單：**標題**、**詳細描述**，選一個**開發強度**（⚡快速 / ⚖️穩健 / 🔬徹底），按 **🚀 開始**。
4. 畫面會跳到**進度頁**：自己更新的進度條，並顯示「⏳ 預計還要：約 N 分鐘」（時間為 UTC+8，會越用越準）。
5. **📝 開工前確認**：AI 規劃完會停下來，用白話告訴你它打算怎麼做。看了沒問題按 **✅ 確認開工**；方向不對按 **❌ 取消**，補充需求後重送即可（不用白等）。
6. **完成**：
   - 全新專案 → 出現「⬇️ 下載你的專案」；若是網頁，還有「👀 線上預覽成果」可直接在瀏覽器看。
   - 現有專案 → 出現「查看成果草案」連結（PR）。不確定就把這個頁面連結傳給工程師。
7. 想看所有任務，點首頁的「查看所有任務 →」。

> 工程師或進階使用者也能用 API 提需求：
> `curl -X POST http://localhost:8080/gateway/issue -H 'Content-Type: application/json' -d @examples/hello-world-issue.json`
> 回應的 `taskId` 對應進度頁 `http://localhost:8080/gateway/ui/<taskId>`。

## 怎麼提一個好需求？

你只需要描述「要什麼」，不用講「怎麼做」。一個好需求包含：

- **標題**：一句話講清楚要做什麼。
- **描述**：背景、想解決的問題、期待的結果。越具體越好。
- **重要性**：P1（很急）、P2（普通）、P3（不急）。

範例：

> **標題**：結帳頁面加上「儲存常用收件地址」
> **描述**：客人每次都要重打地址很麻煩。希望登入後可以儲存最多 3 筆常用地址，結帳時直接選。

## 進度代表什麼？

| 你看到的 | 意思 |
| --- | --- |
| 📩 任務已受理 | 系統收到你的需求了 |
| 🧠 正在構思實作方案 | AI 正在想怎麼做 |
| 📝 請確認開工方向 | 停下來等你看計畫、按確認或取消 |
| 👩‍💻 AI 工程師平行開發中 | 同時在試幾種寫法 |
| 🧪 正在挑選最佳方案 | 從幾個版本選最好的 |
| 📬 已產生成果草案 | 程式碼準備好了 |
| 🔍 程式碼品質與安全檢查 | 自動審查中 |
| 🛠️ 依審查建議修正中 | 依審查意見修正 |
| ✅ 開發完成 | 完成！可以請工程師合併 |
| ⚠️ 發生問題 | 需要請工程師看一下 |

## 安全感：它不會弄壞現有系統

- 改現有專案時，結果都是一個 **Pull Request（草稿）**，標上 `ai-generated`，**一定要人類同意才會合併**；AI **永遠不會**直接改你的正式分支（main / release）。
- 做全新專案時，成果只在你的工作目錄裡（可下載／預覽），不會碰到任何外部系統。
- 開工前一定會先讓你確認方向，做錯也只在你按下「確認」之後才開始。
- 你可以隨時關掉視窗，進度不會中斷。

---

## 遇到問題？

- 進度卡在 ⚠️：把 `taskId` 給工程師，請他看 [SETUP.md](SETUP.md) 的疑難排解。
- 想確認系統設定正不正確：請管理員執行 `bash scripts/doctor.sh`。
