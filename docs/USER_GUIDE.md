# 使用指南（給操作者）

這份文件寫給**不需要懂程式**的人。你只要會「提需求」和「看結果」，AI Factory 會幫你把程式寫好、檢查好，最後交給工程師按一個鍵合併。

---

## 它在幫你做什麼？

你提出一個需求，AI Factory 會自動：

1. 🧠 想好怎麼做（產生實作計畫）
2. 👩‍💻 派幾個 AI 工程師同時嘗試不同寫法
3. 🧪 挑出最好的一版
4. 🔍 自動做程式碼品質與安全檢查
5. 🛠️ 依檢查結果修正
6. ✅ 開一個「Pull Request」，等人類點頭合併

整個過程你都能在一個網頁上看到進度，不用看任何程式碼。

---

## 第一次使用：用網頁提需求（最簡單）

請你的管理員先完成 [SETUP.md](SETUP.md) 的安裝，然後：

1. 打開首頁（管理員會給你網址，通常是 `http://localhost:8080`）。
2. 填寫表單：**標題**、**詳細描述**，選一個**開發強度**（⚡快速 / ⚖️穩健 / 🔬徹底），按 **🚀 開始**。
3. 畫面會自動跳到**進度頁**：一個會自己更新的進度條，並顯示「⏳ 預計還要：約 N 分鐘」（系統會隨著實際完成的任務越來越準）。完成後會出現綠色的「✅ AI 已經完成你要求的工作了」，並有一個按鈕讓你查看成果草案。
4. 想看所有任務，點首頁的「查看所有任務 →」。

> 工程師或進階使用者也能用 API 提需求：
> `curl -X POST http://localhost:8080/gateway/issue -H 'Content-Type: application/json' -d @examples/hello-world-issue.json`
> 回應的 `taskId` 對應進度頁 `http://localhost:8080/gateway/ui/<taskId>`。

---

## 怎麼提一個好需求？

你只需要描述「要什麼」，不用講「怎麼做」。一個好需求包含：

- **標題**：一句話講清楚要做什麼。
- **描述**：背景、想解決的問題、期待的結果。越具體越好。
- **重要性**：P1（很急）、P2（普通）、P3（不急）。

範例：

> **標題**：結帳頁面加上「儲存常用收件地址」
> **描述**：客人每次都要重打地址很麻煩。希望登入後可以儲存最多 3 筆常用地址，結帳時直接選。

## 進度代表什麼？

| 你看到的 | 意思 |
| --- | --- |
| 📩 任務已受理 | 系統收到你的需求了 |
| 🧠 正在構思實作方案 | AI 正在想怎麼做 |
| 👩‍💻 AI 工程師平行開發中 | 同時在試幾種寫法 |
| 🧪 正在挑選最佳方案 | 從幾個版本選最好的 |
| 📬 已產生成果草案 | 程式碼準備好了 |
| 🔍 程式碼品質與安全檢查 | 自動審查中 |
| 🛠️ 依審查建議修正中 | 依審查意見修正 |
| ✅ 開發完成 | 完成！可以請工程師合併 |
| ⚠️ 發生問題 | 需要請工程師看一下 |

---

## 安全感：它不會弄壞現有系統

- 所有結果都是一個 **Pull Request（草稿）**，標上 `ai-generated`，**一定要人類同意才會合併**。
- AI **永遠不會**直接改你的正式分支（main / release）。
- 你可以隨時關掉視窗，進度不會中斷。

---

## 遇到問題？

- 進度卡在 ⚠️：把 `taskId` 給工程師，請他看 [SETUP.md](SETUP.md) 的疑難排解。
- 想確認系統設定正不正確：請管理員執行 `bash scripts/doctor.sh`。
